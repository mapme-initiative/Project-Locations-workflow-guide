# Step 5: Aggregate data

!!! overview
    **Tool:** Python script

    **Performed by:** <span class="glossary-term" data-term="idv">Geodata-Desk</span>

    **Maintained by:** <span class="glossary-term" data-term="idv">Geodata-Desk</span>

    **Previous step:** [4 Upload data](04-upload-data.md)

    **Next step:** [6 Create map](06-create-map.md)

## Purpose

Aggregate multiple project-level locations datasets to a single departmental-level dataset and published as a new <span class="glossary-term" data-term="odp">ODP</span> data resource.

## Requirements

- Validated Project-level locations
- Aggregation Python script (contact <span class="glossary-term" data-term="idv">Geodata-Desk</span>)

## Procedure

1. Confirm the project-level <span class="glossary-term" data-term="odp">ODP</span> data resource is available and current.


2. Run the aggregation Python script.


3. Confirm that the new departmental-level data resource is ready to be used for portfolio map or dashboard creation once uploaded.

## Outputs

| Item | Format | Notes |
|------|--------|-------|
| Departmental-level Project Locations | <span class="glossary-term" data-term="odp">ODP</span> data resource | Basis for creating portfolio maps and/or dashboards |
