# Step 6: Create map

!!! overview
    **Tool:** <span class="glossary-term" data-term="odp">ODP</span>

    **Performed by:** <span class="glossary-term" data-term="idv">Geodata-Desk</span>

    **Maintained by:** <span class="glossary-term" data-term="zdv">ZDV</span> (<span class="glossary-term" data-term="dai">DAI</span>)

    **Previous step:** [5 Aggregate data](05-aggregate-data.md)

## Purpose

Publish either a **project-level locations map** or a **portfolio map** as an <span class="glossary-term" data-term="odp">ODP</span> map resource.

## Requirements

- Project-level Project Locations <span class="glossary-term" data-term="odp">ODP</span> data resource from [Step 4: Upload data](04-upload-data.md) for project-level map
- Departmental-level Project Locations <span class="glossary-term" data-term="odp">ODP</span> data resource from [Step 5: Aggregate data](05-aggregate-data.md) for portfolio map


## Procedure

1. In ODP, create a new map resource using the data resource needed. Geodata-Desk can be contacted for support.


3. Configure the portfolio map display settings as required (e.g. changing symbology, adding filters, etc.). Verify that the map renders correctly for end users and assign access accordingly.

## Outputs

| Item | Format | Notes |
|------|--------|-------|
| Project-level locations map | ODP map resource | Interactive project locations map |
| Departmental-level locations map | ODP map resource | Interactive portfolio map |
