# Step 4: Upload data

!!! overview
    **Tool:** <span class="glossary-term" data-term="odp">ODP</span>

    **Performed by:** KfW operations (<span class="glossary-term" data-term="pmpa">PM/PA</span>)

    **Maintained by:** <span class="glossary-term" data-term="zdv">ZDV</span> (<span class="glossary-term" data-term="dai">DAI</span>)

    **Previous step:** [3 Visual plausibility check](03-visual-plausibility-check.md)

    **Next step:** [5 Aggregate data](05-aggregate-data.md)

## Purpose

Upload the validated GeoJSON of project-level locations to the <span class="glossary-term" data-term="odp">ODP</span> as a data resource, and assign access to colleagues who need to view or work with the dataset.

For all Project Locations, the valid GeoJSON file generated by the validator tool can be uploaded directly to the **<span class="glossary-term" data-term="odp">ODP</span>**, either by the operations team member responsible for updating Project Locations or with support from Geodata-Desk.

## Requirements

- Confirmed validated GeoJSON from [Step 3: Visual plausibility check](03-visual-plausibility-check.md)
- Log into <span class="glossary-term" data-term="odp">ODP</span> at least once to be recognized on the platform with access credentials (i.e. email account from any German institution)
- List of KfW colleagues or partners who should have access to the dataset
- Colleagues must have **signed in to <span class="glossary-term" data-term="odp">ODP</span> at least once** — only users who have accessed the platform before appear in the sharing list

### Requesting support from Geodata-Desk

If support from <span class="glossary-term" data-term="idv">Geodata-Desk</span> is preferred, send an email with the following information:

| Required information | Details |
|---------------------|---------|
| **Attachments** | **Points-based:** validated <span class="glossary-term" data-term="geojson">GeoJSON</span> or unvalidated PLM |
| | **Lines / polygons:** validated <span class="glossary-term" data-term="geojson">GeoJSON</span> or unvalidated PLM **and** geometry files (if not yet successfully joined) |
| **Access list** | Email addresses of <span class="glossary-term" data-term="kfw">KfW</span> colleagues who should have access to the dataset (each must have signed in to <span class="glossary-term" data-term="odp">ODP</span> at least once) |
| **Deadlines** | Any urgent deadlines by which the dataset and/or map need to be prepared |

!!! note "PLM errors"
    If there are errors in the <span class="glossary-term" data-term="plm">PLM</span> that cannot be resolved, contact <span class="glossary-term" data-term="idv">Geodata-Desk</span> for further support.

### Procedure

1. Open the [<span class="glossary-term" data-term="kfw">KfW</span> Open Data Platform](https://opendata.oda.kfw.de/datasets) and sign in. Select **Datasets** from the top navigation menu, then click the **New** button (top right of the dataset list) and choose **Upload dataset** from the dropdown menu.

    <img
      src="../../assets/images/upload-data-datasets-new-menu.png"
      alt="ODP Datasets page — New → Upload dataset"
      class="instruction-figure instruction-figure--90"
    />


2. On the **Upload a dataset** page, click **+ Select files…** or drag and drop your validated <span class="glossary-term" data-term="geojson">GeoJSON</span> into the upload area. Supported formats include GeoJSON, GeoPackage, ESRI Shapefile, KML, CSV, and others listed on the page. Click **Upload** to publish the dataset.

    <img
      src="../../assets/images/upload-data-upload-page.png"
      alt="ODP Upload a dataset page"
      class="instruction-figure instruction-figure--80"
    />


3. Once Project Locations are uploaded, a **link to the data resource** is generated.


4. Confirm the file appears as an **<span class="glossary-term" data-term="odp">ODP</span> data resource** and note the resource location for the [Aggregate step](05-aggregate-data.md).


5. To modify access to the dataset (e.g. share with additional colleagues), select **Resource → Share**. Colleagues need to have signed on to <span class="glossary-term" data-term="odp">ODP</span> **at least once** for their accounts to be visible and assignable.

    *(In German: **Ressource → Teilen**)*

    <img
      src="../../assets/images/upload-data-share-resource-teilen.png"
      alt="ODP Resource Share menu"
      class="instruction-figure instruction-figure--small"
    />


6. A pop-up window appears where permissions can be updated for individuals or groups.

    <img
      src="../../assets/images/upload-data-permissions-popup.png"
      alt="ODP permissions pop-up"
      class="instruction-figure instruction-figure--50"
    />


7. Once each user has been assigned their permissions, **save the dataset**.

    *(In German: **Ressource → Speichern**)*

    <img
      src="../../assets/images/upload-data-save-resource-speichern.png"
      alt="ODP save resource"
      class="instruction-figure instruction-figure--small"
    />
