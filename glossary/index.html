    Glossary¶

    Alphabetical reference for abbreviations used in this guide. Click any highlighted term on a step page to open a quick reminder, or browse the full list below.

    ← Back to workflow overview

    CRS¶

    Coordinate Reference System

    A coordinate-based system for representing locations on Earth. GeoJSON in this workflow must use EPSG:4326 (WGS 84).

    DAI¶

    Team Data Analytics and Artificial Intelligence (within KfW)

    Team that develops and maintains the online validator and Open Data Platform components used in this workflow.

    EPSG:4326¶

    WGS 84 geographic coordinate system

    Standard geographic coordinate reference system (longitude/latitude in decimal degrees). Required CRS when exporting GeoJSON geometries in QGIS for this workflow.

    Geodata-Desk¶

    Team Geodata-Desk (within KfW)

    Maintains the PLM, performs aggregation and map creation, and supports geometry-based data preparation.

    GeoJSON¶

    Geographic JSON

    Open standard file format for encoding geographic features and their properties. Used throughout the workflow for validated project locations and ODP uploads.

    GIS¶

    Geographic Information System

    Software for creating, editing, and analysing spatial data. QGIS is used in the Collect step to draw polygon and line geometries.

    KfW¶

    Kreditanstalt für Wiederaufbau

    German development bank

    ODP¶

    Open Data Platform

    KfW internal platform for curating, visualizing, and publishing geospatial data and map resources. Access outside of KfW is possible for any partners or collaborators with an email address from a German institution.

    PA¶

    Project Assistant

    KfW operations role (with PM) responsible for updating project locations, including visual plausibility checks and ODP uploads.

    PIA¶

    PIA/ external consultant

    External consultant who performs the Collect and Validate steps using the PLM and online validator.

    PLM¶

    Project Locations Model

    Excel template (XLSX) for collecting Project Locations metadata. Joined to GeoJSON geometry files for multi-geometry locations or completed with latitude/longitude columns for point locations.

    PM¶

    Project Manager

    KfW operations role (with PA) responsible for project location updates, including visual plausibility checks and ODP uploads.

    PM/PA¶

    Project Manager/ Project Assistant

    KfW operations roles that perform visual plausibility checks and upload validated data to ODP.

    QGIS¶

    QGIS

    Free, open-source GIS desktop application used to create and export GeoJSON geometries for lines and polygons in the Collect step.

    WGS 84¶

    World Geodetic System 1984

    Global geographic coordinate system (EPSG:4326) required for GeoJSON export in this workflow.

    XLSX¶

    Excel spreadsheet format

    Microsoft Excel Open XML format. The PLM is provided and completed as an XLSX file.

    ZDV¶

    Zentrale Datenverarbeitung (Central Data Processing)

    KfW unit hosting Team DAI, which maintains the online validator and ODP tools used in Validate, Visual check, Upload, and Create map steps.
